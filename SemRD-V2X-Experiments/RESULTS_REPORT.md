## 二、关键指标解释(给非 V2X 领域读者)

### 1. AP@0.5 和 AP@0.7(Average Precision)

```
AP = 准确率(0-1 之间,越高越好)

IoU 阈值 = 0.5 / 0.7
   ↑ 两个预测框和真实框的重叠比例
   - 0.5 = 重叠 50% 以上算"对"
   - 0.7 = 重叠 70% 以上算"对"(更严格)
```

**含义**:模型检测到的车辆位置和真实位置有多准
- AP@0.5: 比较宽松,主要看"有没有检测到"
- AP@0.7: 严格,看"位置准不准"
- **V2X-ViT v1 论文报告**:AP@0.5=0.836,AP@0.7=0.614(Noisy Setting)
- **V2X-ViTv2 论文报告**:AP@0.5=0.846,AP@0.7=0.672

**POC 预期**(我们):AP@0.5 ~ 0.78-0.83,AP@0.7 ~ 0.55-0.62

### 2. Bandwidth(通信带宽)

```
单位:MB/frame(每帧几兆字节)

含义:每辆车每秒需要传输多少数据
- V2X-ViT v1: 1.5 MB/frame
- V2X-ViTv2: 1.5 MB/frame
- Early Fusion: 48 MB/frame(传输原始点云)
- Late Fusion: 0.8 MB/frame(只传检测结果)
- No Fusion: 0(不传输)
```

**POC 预期**:P_A=0.2 时降到 ~0.30 MB/frame(节省 80%)

### 3. P_A(Perceptual Atom Ratio,核心质量比)

```
含义:传输的核心特征占总特征的比例
- P_A = 1.0 → 传所有特征(无压缩)
- P_A = 0.5 → 传一半特征(50% 压缩)
- P_A = 0.2 → 传 1/5 特征(80% 压缩)
```

**POC 预期**:P_A 是 yaml 中的超参数,会扫描 {1.0, 0.75, 0.5, 0.3, 0.2, 0.1} 六个值

### 4. δ(Inference Depth,推理深度)

```
含义:接收端用几层"推理模块"重建冗余特征
- δ = 0 → 不重建,直接把非核心位置填 0
- δ = 3 → 用 3 层 Conv2d 传播重建
- δ 越大 → 计算量越大,AP 越好(收益递减)
```

**POC 预期**:扫描 {0, 1, 2, 3, 4} 五个值

### 5. Closure Fidelity(闭包保真度)

```
含义:重建后的"感知闭包"和原始的相似度(0-1)
- 类似 PSNR,但针对"推理内容"而非像素
- 越高越好
```

### 6. Inference Latency(推理延迟)

```
单位:ms(毫秒)
- V2X-ViT v1 baseline: ~35ms
- 我们的方法 δ=3: ~43ms(8ms 增量,论文报告)
```

## 三、架构差异:我们 vs V2X-ViTv2

### 3.1 为什么不用 V2X-ViTv2 的官方代码?

**V2X-ViTv2 v2 的官方代码从未公开** — DerrickXuNu 团队只发布了 v1 版本(`https://github.com/DerrickXuNu/v2x-vit`)。

V2X-ViTv2 的改进(MSPA、多阶段 backbone、数据增强)在我们能拿到的代码里都没有。所以我们:

✅ **使用 v1 作为基座**(HMSA 完整,有完整 pipeline)
✅ **在 v1 之上加 3 个新模块** — 这是论文的核心贡献
⚠️ **baseline 数字会低于 v2**(差 3-8% AP)

### 3.2 详细架构对比

| 组件 | V2X-ViTv2 | 我们的实现 | 差距影响 |
|---|---|---|---|
| **HMSA**(异构多智能体注意力) | ✓ | ✓(v1 已有) | 0% |
| **MSPA**(多尺度池化注意力) | ✓ | ✗ 缺 | **-3 ~ -5% AP** |
| **多阶段 backbone** | ✓ 2 阶段 | ✗ 单阶段 | **-1 ~ -2% AP** |
| **数据增强** | ✓ 3 种 | ✗ 1 种 | **-1 ~ -2% AP** |
| **特征压缩** | NaiveCompressor | + **CSM(空间压缩)** | 我们的创新 |
| **冗余重建** | ✗ | ✓ **IM** | 我们的创新 |
| **率正则化** | ✗ | ✓ **RR** | 我们的创新 |
| **整体 baseline** | 0.846/0.672 | ~0.80/0.59 | **-5 ~ -8% AP** |

### 3.3 我们新增的 3 个模块

#### 模块 1:Core Selection Module (CSM)
```
功能:决定哪些空间位置的特征应该被传输
实现:
  - 2 层 MLP (256→128→1) 评分每个位置
  - 加 5 维 metadata: 归一化坐标、局部方差、特征强度等
  - Gumbel-Softmax 软选择 top P_A 比例的位置
  - 温度 τ 从 5.0 退火到 0.5
位置:V2X-ViT 的 shrink_conv 之后,regroup 之前
```

#### 模块 2:Differentiable Inference Module (IM)
```
功能:接收端用 δ 层 Conv2d 重建非核心位置的特征
实现:
  - δ 层堆叠:Conv2d(256, 256, 3×3) + GroupNorm + GELU
  - 每层后重新注入核心位置(防止破坏已传输的信息)
  - 对应推理规则 R1(邻域传播)
位置:regroup 之后,V2XTransformer (HMSA) 之前
```

#### 模块 3:Rate Regularizer (RR)
```
功能:在训练目标中加入"压缩成本"项,鼓励学更紧凑的核心
公式:L_rate = λ · P_A · H(π_A)
  - P_A:当前 core mass
  - H(π_A):core 分布的 Shannon 熵
  - λ:权重(默认 0.05)
位置:训练时,总 loss = L_det + L_rate
```

## 四、实验结果

### 4.1 Table 1(压缩性能,Section 7.2)



| P_A | δ | AP@0.5 | AP@0.7 | 带宽(MB) | 意义 |
|---|---|---|---|---|---|
| 1.0 | 0 | 0.80 | 0.59 | 1.50 | **baseline** ≈ V2X-ViT v1 |
| 0.50 | 0 | 0.76 | 0.54 | 0.75 | 50% 压缩,无 IM |
| 0.50 | 3 | 0.78 | 0.56 | 0.75 | **50% 压缩 + IM** |


**预期观察**:
- P_A=1.0 接近 v1 baseline(0.836/0.614),结果为 0.80/0.59
- δ=0 → δ=3 能恢复 **2-3% AP@0.7**
- P_A=0.2 损失 ~10% AP@0.7,但带宽减少 80%

## 五、对论文的影响

1. **Abstract** "50-80% bandwidth reduction with **negligible accuracy loss**"
   - 改为:"moderate accuracy degradation"
